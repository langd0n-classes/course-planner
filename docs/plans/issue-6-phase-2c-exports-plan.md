## Recommended decisions before coding

Two decisions need human confirmation because they materially determine the output contract:

1. **Blackboard document structure.** Confirm whether Blackboard expects a particular institutional heading hierarchy, table layout, or import template. Recommended default: a generic `.docx` with module title/description, learning objectives, and ordered session overview. Do not add Blackboard-specific XML packaging or institution-specific language without an actual requirement.
2. **Private planning notes.** Confirm whether `Module.notes` and `Session.notes` belong in exports. Recommended default: exclude them because Phase 2B labels them planning notes, while exports leave the instructor’s private workspace.

The following can be assumed and recorded in `ASSUMPTIONS.md` unless the issue says otherwise:

- Canceled sessions remain visible in summaries, explicitly marked canceled, because they are part of the planning history.
- Redistributed coverage is represented from the current session/coverage records without exporting internal audit IDs.
- Dates use a deterministic, locale-independent human format.
- Missing optional content produces an omitted section or a concise “not specified” marker, never fabricated text.
- Export filenames are sanitized and derived from stable course/module/session codes rather than UUIDs.
- All three outputs are generated on demand and are not persisted as `Artifact` rows.

## Architecture and design concerns to address explicitly

1. **The existing artifact export subsystem does not cleanly fit this issue.**  
   `src/app/api/artifacts/export/[id]/route.ts` requires a persisted `Artifact` ID, cannot represent a term parent, and uses a mock exporter that returns placeholder markdown. Its module fallback can even pass an artifact ID as a module ID. Phase 2C should use direct resource routes and on-demand output rather than forcing term exports into this model.

2. **Do not interpret “mock AI is intentional” as requiring mock exports.**  
   The session prompt is deterministic text preparation, not AI generation. It should use real persisted data while leaving `MockAiPlanner` unchanged.

3. **Avoid an exports workspace.**  
   A dashboard, export history, template manager, or preview system would conflict with “exports are a failure state.” Three contextual secondary actions are enough.

4. **Do not export private workspace fields by accident.**  
   `Module.notes` and `Session.notes` were added specifically as planning workspace content. Their inclusion requires an intentional human decision.

5. **Avoid duplicate health semantics.**  
   The term summary must reuse the current canceled-session and I/P/A health rules from Phase 2B rather than inventing export-only calculations.

6. **Phase 2B.2 baseline must be reconciled first.**  
   The current branch links to `/terms/[id]/flow`, but the route files are only on the issue #5 feature branch. Phase 2C agents should not absorb that merge/reconciliation into an export chunk; it is a prerequisite owned by #5.

7. **The service interface is broader than Phase 2C.**  
   Notebook skeleton and skills-list exports are outside issue #6. Leave them unexposed and do not polish or expand them as part of this work. If the interface is adjusted, preserve compatibility where practical and document why; do not restructure the whole services layer.
